# Module 1 | Class 4 Assignment: ITES Case Study — Recommendation or Fraud

**Related slides:** Class 4 — Ethics in AI: Bias, Fairness, and Privacy
**Estimated time:** 60 minutes

## Objective
Analyze a real ITES (IT-Enabled Services) ML application from both a technical and ethical angle. Choose one scenario and build a complete analysis.

## What You Need
- Tool: Google Docs or any word processor
- No code required
- Reference: Class 3 notes (ITES case studies) and Class 4 notes (ethics, bias, fairness)

## Choose One Scenario

**Option A: E-Commerce Recommendation System (Uzum-like platform)**
An Uzbek e-commerce platform wants to build a recommendation engine. Users browse thousands of products daily. The platform wants to show personalized suggestions on the homepage and in "you might also like" sections. They have 2 years of user clickstream data, purchase history, and product metadata.

**Option B: Mobile Payment Fraud Detection**
A mobile money service (like Payme or Click) processes millions of transactions monthly. They want an ML system that flags suspicious transactions in real time. They have historical transaction logs, user profiles, and a small set of confirmed fraud cases labeled by their security team.

## Tasks

### Task 1: Problem Type
**What to do:** Identify what kind of ML problem this is (classification, regression, clustering, etc.) and explain why.

**How to do it:**

- Step 1: Review the scenario. Ask yourself: is the model predicting a category (classification), a number (regression), or finding groups (clustering)?
- Step 2: For Option A, consider: is the model classifying items as "relevant/not relevant" or predicting a relevance score? Both are valid — pick one and justify.
- Step 3: For Option B, consider: is the model classifying transactions as "fraud/not fraud" (binary classification)? What makes this tricky (class imbalance — fraud is rare)?

**Tool:** Google Docs

**Help if stuck:** Google ML Crash Course — "Framing" section explains how to define ML problem types: https://developers.google.com/machine-learning/crash-course

**Slide reference:** This was covered in Class 3, Slides: "How Netflix, YouTube, and Uzum Know What You Want" and "Catching Criminals with Machine Learning"

### Task 2: Data Requirements
**What to do:** List at least 8 specific data fields the model would need.

**How to do it:**

- Step 1: Think about what information the model needs to make its prediction.
- Step 2: For Option A (recommendations): user_id, product_id, click_history, purchase_history, product_category, price, time_on_page, search_queries, user_demographics, product_ratings.
- Step 3: For Option B (fraud): transaction_id, amount, timestamp, sender_id, receiver_id, transaction_type, device_info, location, historical_transaction_frequency, account_age.
- Step 4: List each field with a 1-sentence explanation of why it matters.

**Tool:** Google Docs — use a bulleted list

**Slide reference:** This was covered in Class 3, Slide: "What Data Does Each System Need?"

### Task 3: Evaluation Metric
**What to do:** Define how you would measure if the model is working. Explain why you chose that metric over others.

**How to do it:**

- Step 1: For Option A, consider: click-through rate (CTR), conversion rate, mean average precision (MAP), or user engagement time.
- Step 2: For Option B, consider: precision (how many flagged transactions are actually fraud), recall (how many real fraud cases are caught), F1-score, or AUC-ROC.
- Step 3: Explain the tradeoff. For fraud detection: high precision means fewer false alarms but might miss real fraud. High recall catches more fraud but annoys legitimate users with false flags.

**Tool:** Google Docs

**Help if stuck:** Google ML Crash Course — "Classification" section covers precision, recall, and evaluation metrics: https://developers.google.com/machine-learning/crash-course

**Slide reference:** This was covered in Class 2, Slide: "The ML Workflow — Evaluation"

### Task 4: Bias and Fairness Risks
**What to do:** Identify what could go wrong and who might be unfairly affected.

**How to do it:**

- Step 1: Think about who is in the training data and who is not.
- Step 2: Consider Uzbekistan-specific factors: urban/rural divides, age gaps in tech adoption, language differences (Uzbek vs Russian speakers), income disparities.
- Step 3: For Option A: Could the recommendation system create filter bubbles? Could it favor popular products over niche ones, disadvantaging small sellers?
- Step 4: For Option B: Could the fraud model disproportionately flag transactions from certain regions or demographics?

**Tool:** Google Docs

**Help if stuck:** Elements of AI, Chapter 7 (AI ethics and societal implications): https://www.elementsofai.com/

**Slide reference:** This was covered in Class 4, Slides: "AI Bias — It Starts with Us" and "Case Study — Amazon's Resume Screening Tool"

### Task 5: Privacy Concerns
**What to do:** Identify what user data is involved and the risks of collecting it.

**How to do it:**

- Step 1: List the personal data involved (browsing history, financial transactions, location, etc.).
- Step 2: Consider: what happens if this data is breached? What could a bad actor do with it?
- Step 3: Consider Uzbekistan's developing data protection landscape — think about what should be protected even if current law does not require it.

**Tool:** Google Docs

**Slide reference:** This was covered in Class 4, Slide: "Privacy — Your Data, Their Model"

### Task 6: Your Recommendation
**What to do:** Should this system be deployed as-is, with guardrails, or not at all? Justify your position.

**How to do it:**

- Step 1: Weigh the benefits (better user experience, revenue, fraud prevention) against the risks (bias, privacy, errors).
- Step 2: If you say "with guardrails," specify exactly what guardrails. For example: human review of flagged cases, regular bias audits, data anonymization, opt-out mechanisms.
- Step 3: Write 3-5 sentences with a clear recommendation.

**Tool:** Google Docs

**Slide reference:** This was covered in Class 4, Slide: "What Does Fair Even Mean?"

### Task 7: Reflection Questions
**What to do:** Answer these three questions (2-3 sentences each).

1. Would your ethical concerns change if this system were deployed in a country with strict data privacy laws (e.g., EU with GDPR)?
2. Is it possible to build a useful recommendation/fraud system without collecting personal data? What would you lose?
3. Who should be responsible when an ML system makes a wrong decision — the developer, the company, or the algorithm?

**Tool:** Same Google Doc
